# Sub-recipe: cross-border-residency

For items that don't belong to a single entity/jurisdiction, or that are
specifically about tax residency status, day-counts, or how one country's
rules affect filings in another (US, Spain, Indonesia). Use this when Step 1
in the main SKILL.md couldn't pin the document to exactly one entity, and the
user has confirmed it's a residency/cross-border matter rather than asking
you to keep guessing.

## How to read it

- Identify which jurisdictions are actually in play (e.g., "does time spent
  in Indonesia break Spanish tax residency", "does the Wyoming LLC's
  US-source income affect Spanish residency filing").
- Look for concrete triggers: day-counts near a 183-day threshold, a tax
  residency certificate request/expiration, a Double Tax Agreement (DTA)
  reference, an NPWP-related Indonesian item, or a request from one
  country's authority for proof of residency elsewhere.
- These items rarely have a single hard-and-fast deadline the way a single-
  jurisdiction notice does — check for one, but don't invent one if the
  document doesn't state it.

## Urgency rules

- 🔴 Critical: a specific authority (AEAT, IRS, or Indonesian DGT) has
  requested residency proof/certificate by a stated date, or a day-count
  threshold is about to be crossed within days and crossing it changes tax
  status.
- 🟡 Moderate: a residency question exists (e.g., approaching 183 days in a
  jurisdiction this year) but no authority has made a formal request yet.
- 🟢 Low: general awareness item (e.g., "you spent N days in Indonesia this
  year so far") with no filing consequence yet.

## Next action patterns

- If an authority requested a residency certificate: "obtain the requested
  certificate of tax residency from the relevant tax authority and submit it
  by the stated deadline."
- If it's a day-count/threshold concern with no formal request yet: "confirm
  the current day-count in each relevant jurisdiction this tax year and flag
  to a cross-border tax advisor before any threshold is crossed."
- If it genuinely doesn't map to residency either: say so plainly and ask
  the user what kind of item this is — don't force-fit it here just because
  the other three sub-recipes didn't match.

## Documents/info typically needed

- Travel/day-count log for the relevant tax year(s), per country.
- Any existing tax residency certificate(s).
- Identification of every entity potentially affected (Wyoming LLC, Spanish
  SL, YEC, CBS), since a residency change can touch more than one at once.
- Contact for a cross-border tax advisor — this sub-recipe's "next action"
  should generally route to a professional rather than a DIY filing step.
